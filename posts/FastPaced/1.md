# 1. Ownership

Ownership is the core of Rust’s memory model.

## The Three Rules

1. Each value has exactly one owner.
2. There can only be one owner at a time.
3. When the owner goes out of scope, the value is dropped.

---

## Example 1: Move Semantics

```rust
let s1 = String::from("hello");
let s2 = s1;
```

`String` does not implement `Copy`.
Ownership moves from `s1` to `s2`.
`s1` becomes invalid.

---

## Example 2: Copy Types

```rust
let x = 10;
let y = x;

println!("{}", x); // still valid
```

Primitive types like `i32` implement `Copy`, so they are duplicated, not moved.

# 2. Borrowing & References

Borrowing allows access without ownership transfer.

---

## Example 1: Immutable Borrow

```rust
fn print_len(s: &String) {
    println!("{}", s.len());
}

let s = String::from("hello");
print_len(&s);
println!("{}", s); // still valid
```

---

## Example 2: Mutable Borrow

```rust
fn modify(s: &mut String) {
    s.push_str(" world");
}

let mut s = String::from("hello");
modify(&mut s);
```

---

## Rules

* Many immutable references OR
* One mutable reference
* Never both at the same time

---

# 3. Slices

---

